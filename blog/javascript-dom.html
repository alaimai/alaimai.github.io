Now, Let's talk about Arrays, Objects and Function.

Arrays store data in an ordered sequence of elements, each identified by an index. To access data from an array, you use square brackets [] notation with the index of the element you want to retrieve.

const fruits = ['apple', 'pinapple', 'orange', 'banana']; console.log(fruits[2]);

Objects store data in key-value pairs and it is unordered, where each value is associated with a unique key. To access data from an object, you use dot notation (object.property) or square brackets notation (object['property']) with the key of the property you want to retrieve.

const stock = { product: 'whiteCoffee', price: 5.99 }; console.log(stock.product); console.log(stock['price'])

Function is a blocks of code for a specific task which is reusable. It help organize code, improve readability. It allow you to break down a program into smaller, more manageable parts, leading to shorter, more efficient programs.

Think of a function as a machine designed by people to complete a specific task efficiently and repeatedly. Just like machines are built for specific purposes, functions in programming are created to execute specific task or calculations to the overall functionality of the program.

The Document Object Model (DOM) is a programming interface for web documents. Developers can manipulate the DOM using JavaScript to dynamically update the content, structure, and style of web pages.

While working on the JavaScript-cafe project, after completing each function, I made it a habit to immediately call the function to observe its behavior. If something unexpected occurred, I would inspect the DOM to identify and rectify any issues. The DOM acted as a guide, pinpointing errors and their locations, aiding in troubleshooting and ensuring smooth functionality.
