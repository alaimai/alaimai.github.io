In this sprint, I've been learning about Testing, Problem Solving, Pseudocoding, and revisiting some basic techniques covered in sprint 1. During the JavaScript-kata task, I encountered the "undefined" error several times. Despite using the Rubber Ducky method, where I talked through each line of code, I couldn't figure out the issue. I even looked up the syntax of the method I used in the code, but that didn't help either. Feeling a bit frustrated, I searched for "function result undefined," and it turned out I forgot to return the result. As a beginner coder, I've learned that most errors at this stage are due to syntax or typos, rather than algorithms.

In the Super-Fizzbuzz task, I pseudocoded the problem and encountered an issue on how to iterate over every element in the array. While I could have done it manually, I wondered if there was a built-in method available. After searching on Google, I found the forEach method, which made the code cleaner and simpler.
